# Kernel Arch/Platform Layout Follow-up Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bring the repository docs and verification flow up to date with the current kernel layout, where x86 CPU/MMU/context code lives under `kernel/arch/x86/`, PC hardware support lives under `kernel/platform/pc/`, and shared code remains in the common kernel directories.

**Architecture:** The source move has already landed. This follow-up is intentionally narrow: document the current structure accurately, remove stale references to the pre-refactor paths, and verify the moved tree from a clean build. No runtime behavior changes are in scope.

**Tech Stack:** Freestanding C and NASM x86 kernel, GNU Make, Python `unittest`, repo docs under `docs/`.

---


- [ ] **Step 2: Rewrite Milestone 2 in `docs/arm64-port-plan.md` to match the current split**

Replace the existing numbered list under `### Milestone 2 — Build split and architecture boundary`
with this text:

```md
1. Keep x86 CPU/MMU/context/boot code under `kernel/arch/x86/`, including the
   current `boot/`, `mm/`, and `proc/` x86 subtrees.
2. Keep PC-specific hardware support under `kernel/platform/pc/` instead of
   treating ATA, PS/2, and related port-I/O code as generic x86 code.
3. Introduce `kernel/arch/arch.h` as the shared arch API around behavior-level
   seams: CPU interrupt state, timer/clock hooks, console output, IRQ
   registration/dispatch, TLB invalidation, address-space switching, trap
   entry, and user-mode entry.
4. Make `kernel/lib/klog.c` portable by separating timestamp source, console
   sink, and optional debug sink from the current x86/PC-specific `clock.h`,
   `print_string()`, and debugcon path assumptions.
```
